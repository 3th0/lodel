Choix de la configuration.

LODEL a besoin d'accéder à des scripts dans un répertoire inclus automatiquement ou dont le chemin absolu est connu et fixe.

Option 1 et 2

Si le chemin d'accès absolu à LODEL est connu et fixe précisez le ci-dessous.
Sinon, LODEL a besoin d'un répertoire inclus automatiquement (variable de configuration "include_path" de php). Si c'est le cas : veuillez choisir la deuxième option et préciser le chemin relatif vers ce répertoire.

Erreur: Impossible d'accéder aux scripts par le répertoire . Erreur: Le répertoire "" n'existe pas et il n'est pas possible de le créer.

Option 1

Verifiez le chemin entré ci-dessous. Sinon vérifiez les droits d'écriture dans le répertoire parent du répertoire "".

Option 2

Créez manuellement le répertoire "" et copiez-y les fichiers du répertoire lodel/scripts. Positionnez les droits pour que le serveur puisse accéder aux fichiers en lecture.

Erreur: Impossible de copier les scripts dans le répertoire .

Option 1

Verifiez le chemin entré ci-dessous. Sinon vérifiez les droits d'écriture dans le répertoire parent du répertoire "".

Option 2

Copiez les fichiers du répertoire lodel/scripts dans le répertoire"". Positionnez les droits pour que le serveur puisse accéder aux fichiers en lecture.

Erreur: L'inclusion automatique dans le répertoire ne fonctionne pas. Vérifiez le répertoire entré ci-dessous ainsi que la configuration de php. Si le problème persiste... il n'y a pas grand chose à faire. Attention: Le chemin indiqué ci-dessous a été "deviné" par LODEL. Il n'est pas nécessairement correct et, surtout, il n'est pas nécessairement fixe, c'est à dire qu'il peut être changé sans préavis par l'herbergeur ou par l'administrateur système. Un changement de ce répertoire rendrait l'ensemble du site inaccessible. En cas de doute utilisez une autre option.
