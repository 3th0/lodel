Configuration de la base de donnée.

Option 1

Choisir une base de données ou créez en une nouvelle (si vous avez les droits). Cette base de donnée sera utilisée pour tous les sites si vous cochez la case "base de données unique". Au contraire, si cette case n'est pas cochée, chaque site aura une base de données propre (à condition d'avoir les droits pour les créer) dont le nom sera dérivé de la base de données principale. Le préfix des tables est surtout utile lorsque vous ne pouvez disposez que d'une base de données pour toutes vos applications (cas fréquent chez les hébérgeurs) dans ce cas le préfix permet de distinguer les tables LODEL de celles des autres applications (préfix a priori le mieux approprié: lodel_).

Option 2

Vous pouvez utiliser une base de donnée existante. Cependant, il est préférable de créer un base de donnée dédiée à Lodel:

CREATE database ;

Dans tous les cas, il faut donner les droits "ALL" sur cette base à l'utilisateur "":

GRANT ALL ON .* TO ;
